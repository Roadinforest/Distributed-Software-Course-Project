# 系统架构设计文档

## 1. 架构总览

本系统采用 **微服务架构**，将电商系统核心业务拆分为四个独立服务，每个服务拥有独立的代码仓库、数据库和部署单元。

## 3. 服务拆分原则

### 3.1 拆分依据

| 原则 | 说明 |
|------|------|
| **单一职责** | 每个服务只负责一个业务领域 |
| **数据自治** | 每个服务独立管理自己的数据库，不跨库访问 |
| **独立部署** | 各服务可以独立编译、打包、部署 |
| **松耦合** | 服务间通过 RESTful API 通信，降低依赖 |

### 3.2 各服务职责

#### 用户服务 (User Service) — :8081

| 功能 | 说明 |
|------|------|
| 用户注册 | 新用户账号注册，支持用户名+密码方式 |
| 用户登录 | 登录验证，成功后签发 JWT Token |
| Token 验证 | 校验 JWT Token 有效性，供其他服务调用 |
| 用户信息管理 | 查看、修改个人信息 |

#### 商品服务 (Product Service) — :8082

| 功能 | 说明 |
|------|------|
| 商品管理 | 商品的创建、修改、删除、上下架 |
| 商品查询 | 支持按ID查询、分页列表、关键词搜索 |
| 分类管理 | 商品分类的增删改查 |

#### 订单服务 (Order Service) — :8083

| 功能 | 说明 |
|------|------|
| 创建订单 | 用户下单，调用商品服务和库存服务 |
| 订单查询 | 按订单ID或用户ID查询订单 |
| 订单状态管理 | 待支付→已支付→已发货→已完成→已取消 |
| 订单列表 | 分页查询用户的订单列表 |

#### 库存服务 (Stock Service) — :8084

| 功能 | 说明 |
|------|------|
| 库存查询 | 查询指定商品的库存数量 |
| 库存扣减 | 下单时扣减库存（需保证原子性） |
| 库存补充 | 入库操作，增加库存数量 |

```
┌────────────┐       查询商品信息       ┌──────────────┐
```

### 调用说明

| 调用方 | 被调用方 | 场景 | 方式 |
|--------|---------|------|------|
| 订单服务 | 商品服务 | 下单时查询商品信息和价格 | HTTP GET |
| 订单服务 | 库存服务 | 下单时扣减库存 | HTTP POST |
| 订单服务 | 库存服务 | 取消订单恢复库存 | HTTP POST |
| API Gateway | 用户服务 | 请求到达时验证 JWT Token | HTTP GET |

## 5. 分层架构（单服务内部）

每个微服务内部采用经典的三层架构：

```
┌─────────────────────────────────────┐
│            Handler 层               │  ← HTTP 请求处理、参数校验、响应封装
│         (Gin Router/Handler)        │
├─────────────────────────────────────┤
│            Service 层               │  ← 业务逻辑处理、事务管理
│        (Business Logic)             │
├─────────────────────────────────────┤
│          Repository 层              │  ← 数据库访问、CRUD 操作
│          (GORM / MySQL)             │
├─────────────────────────────────────┤
│            Model 层                 │  ← 数据模型定义
│        (Struct / Entity)            │
└─────────────────────────────────────┘
```

| 层级 | 对应包 | 职责 |
|------|--------|------|
| Handler | `internal/handler/` | 路由注册、请求参数解析与验证、调用 Service 层、封装 HTTP 响应 |
| Service | `internal/service/` | 核心业务逻辑、事务控制、调用 Repository 层及其他服务 |
| Repository | `internal/repository/` | 数据库 CRUD 操作、SQL 查询封装 |
| Model | `internal/model/` | 数据表映射的 Go 结构体、请求/响应 DTO |
| Middleware | `internal/middleware/` | JWT 认证、日志记录、跨域处理、错误恢复 |
| Config | `config/` | 配置文件读取与管理 |

## 7. 分布式缓存架构

### 7.1 Redis缓存策略

```
┌─────────────────────────────────────────────────────┐
│                   Redis 缓存集群                     │
│                                                     │
│   ┌─────────────────────────────────────────┐       │
│   │          Cache Aside 模式               │       │
│   │                                          │       │
│   │  1. 先查缓存 → 命中直接返回               │       │
│   │  2. 未命中 → 查数据库 → 写入缓存          │       │
│   │  3. 更新/删除 → 清除缓存                  │       │
│   └─────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────┘
```

### 7.2 缓存问题解决方案

| 问题 | 解决方案 |
|------|---------|
| **缓存穿透** | 1. 参数合法性校验 2. 空值缓存(短TTL) |
| **缓存击穿** | 分布式锁(SetNX)保护并发查询 |
| **缓存雪崩** | TTL随机抖动 + 热点数据预热 |

### 7.3 缓存配置

```yaml
cache:
  ttl: 300           # 基础TTL(秒)
  max_ttl: 600       # 最大TTL(秒)
  null_ttl: 60       # 空值缓存TTL(秒)
  enable_random: true  # 启用随机TTL防止雪崩
```

## 8. 读写分离架构

### 8.1 MySQL主从复制

```
┌─────────────────────────────────────────────────────┐
│              MySQL 读写分离集群                      │
│                                                     │
│     ┌──────────────┐        ┌──────────────┐       │
│     │  Master      │◄───────│   Slave      │       │
│     │  (主库/写)    │ 复制    │   (从库/读)  │       │
│     │  :3308       │───────►│   :3309       │       │
│     └──────────────┘        └──────────────┘       │
│           │                         │               │
│           ▼                         ▼               │
│     CREATE/UPDATE/DELETE      SELECT               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 8.2 主从复制配置

| 参数 | Master | Slave |
|------|--------|-------|
| server-id | 1 | 2 |
| log-bin | mysql-bin | - |
| read-only | OFF | ON |
| gtid-mode | ON | ON |

### 8.3 代码读写分离实现

```go
// 写操作 → 主库
func (r *ProductRepository) Create(product *model.Product) error {
    return r.db.Create(product).Error  // db = master
}

// 读操作 → 从库
func (r *ProductRepository) FindByID(id uint) (*model.Product, error) {
    return r.dbRead.First(&product, id).Error  // dbRead = slave
}
```

### 8.4 测试接口

- `GET /api/v1/db-stats` - 查看读写分离配置状态
- 日志中可查看读写分离执行记录
